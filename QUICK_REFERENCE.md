## 🏗️ Architecture

```
Frontend → Nginx Gateway → Backend Services
                ↑
                └── JWT Validation here or in services
```

**Current Implementation:**
- Token generated by: **User Service**
- Token validation: **Each Backend Service** (with JwtGuard)
- Public keys: **User Service JWKS endpoint**

## 🔐 Token Info

- **Algorithm**: RS256 (RSA with SHA-256)
- **Key Size**: 2048-bit
- **Expiration**: 30 days
- **Storage**: `keys.json` in user-service container
- **Public Keys**: `/.well-known/jwks.json`

## ⚡ Add JWT to a Service (5 min)

### 1. Copy JWT Guard
```bash
cp services/user-service/src/common/guards/jwt.guard.ts \
   services/lending-service/src/common/guards/
```

### 2. Import & Apply Guard
```typescript
import { JwtGuard } from 'src/common/guards/jwt.guard';
import { UseGuards } from '@nestjs/common';

@Controller('api/lendings')
export class LendingsController {
  @UseGuards(JwtGuard)
  @Get()
  async getAll(@Req() req: Request) {
    // Access user info
    console.log(req.user.user_id);
    return this.service.findAll();
  }
}
```

### 3. Test
```bash
curl -X GET http://localhost/api/lendings \
  -H "Authorization: Bearer $TOKEN"
```

## 🚨 Common Issues

| Error | Solution |
|-------|----------|
| Missing Authorization header | Add `Authorization: Bearer <token>` header |
| Invalid token format | Use `Bearer token`, not just `token` |
| Token signature failed | Ensure user-service is running |
| ECONNREFUSED | Check docker-compose networking |
| Token expired | Login again to get new token |

## 💡 Key Concepts

**JWT (JSON Web Token):**
- Self-contained token with claims
- Can be verified without contacting server
- Signature proves authenticity

**RSA256:**
- Asymmetric encryption (public/private keys)
- Server signs with private key
- Client verifies with public key
- More secure than symmetric (HMAC)

**JWKS (JSON Web Key Set):**
- Public keys in standard JSON format
- Endpoint: `/.well-known/jwks.json`
- Used by clients to verify tokens
- Cached for performance

**JwtGuard:**
- NestJS middleware
- Protects routes
- Validates tokens automatically
- Extracts user claims
